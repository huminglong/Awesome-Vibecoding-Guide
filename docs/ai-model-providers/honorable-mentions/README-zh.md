# 荣誉提及 - AI模型提供商

## 概述
本部分涵盖我使用过但目前不推荐作为主要开发解决方案的AI模型提供商和服务。这些工具和服务有其位置——特别是对于学习、实验或特定用例——但它们具有重大限制，使它们不适合严肃的专业开发工作。

## 为什么是"荣誉提及"？
这里列出的每个提供商都有一些有价值的东西，无论是：
- 预算有限的初学者的优秀学习平台
- 访问多样化的模型选择以进行实验
- 进入AI辅助编码的高性价比方式
- 对特定非生产场景有用

然而，它们都有共同的限制，阻止我推荐它们用于专业开发：
- 可靠性和正常运行时间问题
- 性能限制
- 成本价值比问题
- 有更好的替代方案可用

## 提供商

### 预算和实验平台
- **[Chutes.ai](chutes.ai-zh.md)** - 极其便宜但不可靠；适合学习，不适合生产
- **[OpenRouter](openrouter-zh.md)** - 具有免费层级的大量模型选择；非常适合探索但可靠性不一致
- **[nanoGPT](nanogpt-zh.md)** - 慷慨的订阅限制和广泛的模型多样性，但由于第三方路由存在重大隐私问题；适合作为备份或用于非敏感项目

### 开发平台
- **[Mistral AI](mistral-ai-zh.md)** - 开源冠军，具有独特的"实验计划"，提供免费API访问以换取训练数据。
- **[Factory AI](factory-ai-zh.md)** - 高性价比平台，具有智能缓存和Droid CLI集成；良好的令牌效率，但不推荐作为主要提供商

### 有注意事项的高级服务
- **[Claude Subscription](claude-subscription-zh.md)** - 杰出的模型质量，但对于常规开发来说成本结构令人望而却步

## 一般建议
这些服务可以作为以下方面的优秀起点：
- **学习**：在没有重大财务承诺的情况下了解AI辅助开发
- **实验**：尝试不同的模型和方法
- **探索**：在投资生产解决方案之前找到适合您工作流程的内容

**然而**，如果您：
- 专业编码或为客户编码
- 处理生产应用程序
- 依赖一致的输出质量
- 无法容忍频繁的服务中断

...那么投资来自主要[AI模型提供商](../README-zh.md)部分的可靠、生产级服务。与生产力和可靠性收益相比，成本差异可以忽略不计。

## 我的经验
我个人使用过这些服务中的每一个，并在我的开发旅程的不同阶段发现了它们的价值。这里记录的见解和注意事项来自真实世界的使用，而不仅仅是理论上的担忧。每一个都教会了我一些有价值的东西——即使那个教训是"当您的生计依赖于它们时，投资于可靠的工具"。
