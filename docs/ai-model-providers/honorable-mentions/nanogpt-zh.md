# nanoGPT

## 概述
nanoGPT是一个AI模型提供商，在单个订阅下聚合对许多顶级开源模型的访问。虽然它以有吸引力的价格点提供慷慨的速率限制和广泛的模型多样性，但由于通过第三方提供商路由请求而产生的重大隐私问题使其不适合敏感的开发工作。

## 主要优势
- **慷慨的速率限制**：$8订阅每月60,000次请求
- **广泛的模型选择**：访问各种顶级开源模型
- **实惠的入门点**：实验不同模型的高性价比方式
- **广泛的模型多样性**：非常适合在选择提供商之前测试和比较不同的LLM
- **推荐折扣**：通过[推荐链接](https://nano-gpt.com/invite/9yrQteBR)提供5%折扣

## 关键限制
- **隐私问题**：将请求路由到第三方提供商，而不是基于美国的基础设施
- **数据安全不确定性**：隐私取决于底层模型提供商的政策
- **非自托管**：与Synthetic.new等提供商不同，模型不在nanoGPT自己的基础设施上托管
- **训练数据风险**：无法保证您的代码不会被底层提供商用于训练
- **复杂的提供商链**：多个方参与请求处理增加了安全表面积

## 隐私政策警告
根据nanoGPT的隐私政策：
- 他们声称不存储或使用您的数据
- **然而**，他们明确声明数据处理取决于底层模型提供商
- 这对实际数据保护造成了重大不确定性

## 最佳用例
- **备份提供商**：当主要提供商不可用时的良好回退选项
- **模型探索**：在选择主要提供商之前测试不同模型的绝佳选择
- **非敏感项目**：适合公共项目或学习练习
- **预算开发**：适用于数据隐私不关键的爱好项目
- **模型比较**：用于了解各种开源模型的能力

## 谁应该避免
- **专业开发**：不推荐用于具有敏感数据的客户或商业工作
- **隐私意识的开发人员**：任何处理专有代码或敏感信息的人
- **生产应用程序**：需要明确数据处理和隐私保证的团队
- **企业使用**：具有严格数据治理要求的组织
- **合规敏感项目**：受GDPR、HIPAA或类似法规约束的项目

## 推荐
nanoGPT占据特定的利基：
- **作为备份计划**：当您的主要提供商遇到停机时有用
- **用于模型探索**：非常适合实验不同模型
- **学习目的**：适合教育项目和教程
- **公共项目**：适合代码无论如何都是公开的开源工作

**关键警告**：如果以下情况，请**不要**使用nanoGPT：
- 您正在处理专有或敏感代码
- 您的工作涉及客户数据或机密信息
- 您对代码用于训练有任何担忧
- 您需要明确的数据处理保证以进行合规

**底线**：慷慨的订阅限制和模型多样性使nanoGPT对实验具有吸引力，但隐私权衡使其不适合具有敏感数据的专业开发。除非您特别对数据处理的不确定性和代码用于训练的可能性感到满意，否则选择具有明确基础设施所有权和隐私保证的提供商。

对于大多数开发人员，在Synthetic.new或GLM Coding Plan上花费稍多是值得的，以获得关于数据隐私和安全的安心。
