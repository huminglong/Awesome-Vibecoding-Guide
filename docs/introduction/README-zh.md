在深入了解具体工具和技术之前，重要的是了解我对工具选择和工作流程设计的方法。我的哲学专注于在成本、可靠性、集成和性能之间找到合适的平衡。

- [**阅读完整哲学**](./philosophy-zh.md) - 我选择工具和构建可持续开发工作流程的框架

关键要点：通常，多个专业化、高性价比工具的组合优于单一昂贵的"一体化"解决方案。

## 商业模式和技术栈 💼🧰

- 我运营一个简单、可重复的商业模式：通过社交媒体或直接推荐找到潜在客户，并交付快速、精美的网站。
- 目标客户：小型本地企业（电工、水管工、理发师等）——他们中的许多人要么有一个笨重的WordPress网站，要么根本没有网站。
- 我的提议：提供轻量级、现代化的静态网站，作为大多数小型企业WordPress安装的更好替代方案——更简单、更快、托管和维护成本更低。
- 我使用的核心栈：
  - Astro — 超级容易获得出色的Core Web Vitals分数并交付最少的客户端JS。
  - Tailwind — 我的首选样式框架：编写快速、一致且高度高效。
  - Cloudflare Pages — 极快、免费（对于静态网站），非常适合托管Astro输出。
- 为什么这个栈适合小型企业：
  - 静态Astro网站在最大化性能的同时最小化复杂性和托管成本。
  - Tailwind保持设计迭代快速且可维护。
  - Cloudflare Pages为静态内容提供近乎即时的全球交付。
- 维护和更新：
  - 静态网站确实需要代码级别的更新，但大多数小型客户很少需要频繁更改。
  - 简单的编辑（电话号码、价格、新照片）很快——通常约10分钟——可以通过本地/远程AI代理自动化或完成。
- SEO和成功指标：
  - 专注于性能（目标是在性能上获得85+分，在其他PageSpeed指标上获得100分）、良好的用户体验、结构化数据和适当的meta标签——而不仅仅是关键词堆砌。
  - 一个构建良好、高性能的网站通常会转化并带来推荐——一个满意的客户往往会带来更多客户。
- 一句话推介：快速、现代、低维护的网站，实际表现良好——对于大多数小型企业来说，比笨重的WordPress设置更合适。
- 另外**请记住**，您可能可以以与典型WordPress网站（500-1000美元范围）相当的价格销售此类网站——因为使用vibecoding——开发是无缝的，并且使用适当的栈——我在这个仓库中讨论的——就像与您的项目聊天一样（而且它不贵，所以在您第一次工作300美元以上后，它就为您一年的栈付费了）
