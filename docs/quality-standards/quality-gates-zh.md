# 质量门：发布前检查清单

在发布任何客户项目之前运行的快速检查清单。这些门确保专业质量而不减慢您的工作流程。

## 为什么质量门很重要

**对于您的业务：**
- 防止令人尴尬的错误
- 减少修订请求
- 维护专业声誉
- 证明您的[定价](../business-model/pricing-economics-zh.md)合理

**对于您的工作流程：**
- 交接前5-10分钟检查
- 捕获95%的常见问题
- 系统性方法防止遗忘项目
- 建立客户信任

**规则**：没有项目在不通过关键7的情况下发布。特定门取决于类型。

---

## 关键7（不可协商）

修复这些或不要发布。这些是交易破坏者。

### 1. 所有交互元素工作 ✅

**要测试什么：**
```
✓ 所有按钮触发操作
✓ 所有表单成功提交
✓ 所有链接转到正确的目的地
✓ 所有社交媒体图标正确链接（或被删除）
✓ 模态框打开和关闭
✓ 手风琴/下拉菜单展开和折叠
✓ 导航菜单在移动设备上工作
✓ 搜索工作（如果存在）
✓ 联系表单传递消息
```

**如何测试：**
- 单击每个按钮
- 填写并提交每个表单
- 打开每个模态框/下拉菜单
- 尝试每个链接

**常见失败：**
- 链接到"#"或example.com的社交图标
- 没有后端的联系表单
- 移动设备上损坏的导航
- 无法关闭的模态框

**另请参阅**：[设计一致性 - 功能元素](./design-consistency-zh.md#功能交互元素)

---

### 2. 移动响应（最小375px）✅

**要测试什么：**
```
✓ 无水平滚动
✓ 文本可读而无需缩放
✓ 按钮可点击（不太小）
✓ 图像正确缩放
✓ 导航工作（汉堡菜单功能）
✓ 表单在移动设备上可用
✓ 内容不溢出容器
```

**如何测试：**
```
1. Chrome DevTools（F12）
2. 切换设备工具栏（Ctrl+Shift+M）
3. 设置为375px宽度（iPhone SE）
4. 滚动整个页面
5. 测试所有交互
```

**快速移动检查：**
- 英雄部分适合而无需滚动
- 导航可访问
- CTA按钮可见和可点击
- 联系信息易于点击
- 表单具有适当的输入类型（tel、email）

**常见失败：**
- 固定宽度容器
- 文本太小
- 按钮太靠近
- 具有微小输入的表单
- 未优化的图像导致加载缓慢

**另请参阅**：[性能 - 移动优化](./performance-zh.md)
