# 阶段1：规划（开发前）🗺️

**工具**：任何AI聊天（ChatGPT、Claude、Gemini等）+ Clavix

## 现代规划工作流程

规划是vibecoding真正开始的地方。推荐的方法使用外部AI聊天来完善您的想法，然后利用Clavix生成具有可实施任务列表的综合PRD。

---

## 步骤1：外部聊天构思（主要方法）

在接触任何代码或工具之前，使用任何AI聊天平台来探索和验证您的想法。

### 为什么首先使用外部聊天？

- **思想完善**：自然对话有助于澄清模糊的想法
- **功能验证**：AI帮助识别缺失的要求或边缘情况
- **技术方向**：获得架构和技术栈选择的反馈
- **PRD生成**：以综合描述结束对话，准备好用于Clavix

### 推荐平台

选择您喜欢的任何AI聊天：
- **ChatGPT** - 非常适合一般构思和功能规划
- **Claude**（网络） - 非常适合技术架构讨论
- **Gemini** - 具有强大推理能力的良好替代方案
- **任何其他AI聊天** - 使用最适合您的

### 构思过程

**专业提示**：为了在构思期间进行有效的AI协作，请参阅[掌握AI提示](../prompting/README-zh.md) - 学习如何构建您的问题和请求以获得更好的结果。

**1. 从问题开始**
```
"我想为远程团队构建一个任务管理应用程序。
用户需要协作任务、跟踪时间和生成报告。"
```

**2. 迭代和完善**
- 讨论功能和优先级
- 验证技术方法
- 考虑边缘情况和用户流程
- 讨论架构决策

**3. 早期包括基础设施**

**关键**：在对话期间提及您的技术栈：
```
"我正在考虑使用Astro作为前端，Tailwind作为样式，
并部署到Cloudflare Pages。您怎么看？"
```

这有助于AI（以及后来的Clavix）理解您正在构建**什么**以及**如何**构建它。

**4. 最后获得PRD**

一旦您完善了想法，请聊天生成综合描述：
```
"基于我们的对话，您能创建一个详细的产品需求文档（PRD），包括：
- 项目概述和目标
- 核心功能和功能
- 技术栈（Astro、Tailwind、Cloudflare Pages）
- 用户流程和要求
- 架构考虑

以可用于项目规划工具输入的格式化它。"
```

---

## 步骤2：Clavix PRD生成（必须拥有）

Clavix对于将您的想法转换为具有可实施任务列表的结构化PRD至关重要。

### 为什么选择Clavix？

- 使用CLEAR框架（简洁、逻辑、明确、自适应、反思）以获得最佳结构
- 自动生成综合产品需求文档
- 创建具有明确验收标准的可实施任务列表
- 包括内置的任务跟踪和进度管理
- 优化提示以获得最大的AI效果

### 使用Clavix

**1. 准备您的输入**

使用您聊天对话中的PRD。确保它包括：
- ✅ 您正在构建什么（功能、要求）
- ✅ 如何构建它（技术栈、基础设施、部署）
- ✅ 任何特定约束或偏好

**带有基础设施的示例输入：**
```markdown
# 任务管理平台

使用以下内容构建协作任务管理应用程序：
- **前端**：Astro + React组件
- **样式**：Tailwind CSS
- **后端**：Cloudflare Workers + D1数据库
- **部署**：Cloudflare Pages
- **身份验证**：Cloudflare Access

功能：
1. 用户身份验证和团队工作空间
2. 任务创建、分配和跟踪
3. 每个任务的时间跟踪
4. 每周/每月报告
...
```

**2. 运行Clavix**

```bash
# 全局安装Clavix
npm install -g clavix

# 在您的项目中初始化
clavix init

# 生成带有任务列表的PRD
/clavix:prd
# 或通过CLI：clavix prd

# 生成实施任务
clavix tasks generate
```

**3. 审查输出**

Clavix生成带有可实施任务的结构化PRD，准备好用于开发。

## 步骤3：传统规划步骤

在Clavix生成您的PRD和任务列表后，您将拥有：

### 1. 产品需求文档（PRD）
详细的功能描述和要求

### 2. 架构文档
技术决策和系统设计

### 3. 功能规范
每个功能分解为：
- 用户故事
- 验收标准
- 技术考虑

### 4. 任务分解
功能分解为可操作的开发任务

## 常见问题和故障排除

### Clavix没有生成有用的输出
**问题**：Clavix产生通用或不完整的PRD结构
- **解决方案**：确保您的输入包括什么（功能）和如何（技术栈、基础设施）
- **提示**：使用`/clavix:deep`对复杂项目进行综合分析
- **参见**：[步骤2：Clavix PRD生成](#步骤2clavix-prd生成必须拥有)以获得正确的输入格式

### AI聊天对话偏离轨道
**问题**：构思对话变得不专注或压倒性
- **解决方案**：使用[掌握AI提示](../prompting/README-zh.md)中的结构化提示
- **提示**：从问题开始，而不是解决方案
- **分解**：将大功能分解为更小、可管理的部分

### 项目范围不明确
**问题**：无法为项目定义清晰的边界
- **解决方案**：审查阶段0中的[项目范围](#项目范围)部分
- **从**：MVP（最小可行产品）方法开始
- **另见**：[商业模式规划](../business-model/README-zh.md)以进行客户范围界定

### 规划花费太长时间
**问题**：在规划上花费数天而没有开始开发
- **解决方案**：将规划时间限制在最多1-2天
- **记住**：计划将在开发过程中演变 - 不要过度优化
- **何时前进**：当您有清晰的PRD、架构和定义的第一个功能时

**寻求额外帮助**：查阅[故障排除指南](../troubleshooting/README-zh.md)以获得规划挑战的详细解决方案。
