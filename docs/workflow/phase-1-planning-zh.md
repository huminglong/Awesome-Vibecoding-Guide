# 阶段1：规划（开发前）🗺️

**工具**：任何AI聊天（ChatGPT、Claude、Gemini等）+ Clavix

## 现代规划工作流程

规划是vibecoding真正开始的地方。推荐的方法使用外部AI聊天来完善您的想法，然后利用Clavix生成具有可实施任务列表的综合PRD。

---

## 步骤1：外部聊天构思（主要方法）

在接触任何代码或工具之前，使用任何AI聊天平台来探索和验证您的想法。

### 为什么首先使用外部聊天？

- **思想完善**：自然对话有助于澄清模糊的想法
- **功能验证**：AI帮助识别缺失的要求或边缘情况
- **技术方向**：获得架构和技术栈选择的反馈
- **PRD生成**：以综合描述结束对话，准备好用于Clavix

### 推荐平台

选择您喜欢的任何AI聊天：
- **ChatGPT** - 非常适合一般构思和功能规划
- **Claude**（网络） - 非常适合技术架构讨论
- **Gemini** - 具有强大推理能力的良好替代方案
- **任何其他AI聊天** - 使用最适合您的

### 构思过程

**专业提示**：为了在构思期间进行有效的AI协作，请参阅[掌握AI提示](../prompting/README-zh.md) - 学习如何构建您的问题和请求以获得更好的结果。

**1. 从问题开始**
```
"我想为远程团队构建一个任务管理应用程序。
用户需要协作任务、跟踪时间和生成报告。"
```

**2. 迭代和完善**
- 讨论功能和优先级
- 验证技术方法
- 考虑边缘情况和用户流程
- 讨论架构决策

**3. 早期包括基础设施**

**关键**：在对话期间提及您的技术栈：
```
"我正在考虑使用Astro作为前端，Tailwind作为样式，
并部署到Cloudflare Pages。您怎么看？"
```

这有助于AI（以及后来的Clavix）理解您正在构建**什么**以及**如何**构建它。

**4. 最后获得PRD**

一旦您完善了想法，请聊天生成综合描述：
```
"基于我们的对话，您能创建一个详细的产品需求文档（PRD），包括：
- 项目概述和目标
- 核心功能和功能
- 技术栈（Astro、Tailwind、Cloudflare Pages）
- 用户流程和要求
- 架构考虑

以可用于项目规划工具输入的格式化它。"
```

---

## 步骤2：Clavix PRD生成（必须拥有）

Clavix对于将您的想法转换为具有可实施任务列表的结构化PRD至关重要。

### 为什么选择Clavix？

- 使用CLEAR框架（简洁、逻辑、明确、自适应、反思）以获得最佳结构
- 自动生成综合产品需求文档
- 创建具有明确验收标准的可实施任务列表
- 包括内置的任务跟踪和进度管理
- 优化提示以获得最大的AI效果

### 使用Clavix

**1. 准备您的输入**

使用您聊天对话中的PRD。确保它包括：
- ✅ 您正在构建什么（功能、要求）
- ✅ 如何构建它（技术栈、基础设施、部署）
- ✅ 任何特定约束或偏好

**带有基础设施的示例输入：**
```markdown
# 任务管理平台

使用以下内容构建协作任务管理应用程序：
- **前端**：Astro + React组件
- **样式**：Tailwind CSS
