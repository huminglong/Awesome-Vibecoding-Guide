# 掌握AI编码提示 🎯

**有效AI辅助开发通信的综合指南**

---

## 为什么这很重要

**您的提示技能直接决定您的vibecoding速度和代码质量。**

```
糟糕的提示：10次迭代 × 5分钟 = 浪费50分钟
好的提示：1-2次迭代 × 2分钟 = 总共4分钟

节省的时间：每个功能46分钟
节省的成本：减少95%的令牌
提高的质量：第一次输出就是生产就绪的
```

**本指南教您：** 如何编写在第一次或第二次尝试就能获得高质量代码的提示。

## 常见场景

### "我的AI生成有错误的代码"

**→ 可能原因**：需求模糊，缺少上下文

**修复**：
1. 阅读[基础：常见反模式](./foundations-zh.md#常见反模式)
2. 使用[模板1：功能添加](./template-library-zh.md#模板1简单功能添加)
3. 添加具体的成功标准

---

### "需要10次迭代才能获得可工作的代码"

**→ 可能原因**：初始提示中细节不足

**修复**：
1. 阅读[基础：4组件框架](./foundations-zh.md#好提示的剖析)
2. 研究[任务特定模式](./task-specific-patterns-zh.md)中的示例
3. 使用[自我纠正提示](./advanced-techniques-zh.md#自我纠正提示)

---

### "AI不遵循我的项目约定"

**→ 可能原因**：缺少项目上下文

**修复**：
1. 阅读[高级技术：上下文感知提示](./advanced-techniques-zh.md#上下文感知提示)
2. 创建`.cursorrules`文件（指南中有示例）
3. 明确引用现有模式

---

### "提示太长，浪费令牌"

**→ 解决方案**：优化技术

**修复**：
1. 阅读[高级技术：提示优化](./advanced-techniques-zh.md#提示优化)
2. 使用[文档优先方法](./foundations-zh.md#策略2文档优先方法)
3. 创建可重用模板

---

### "复杂功能让AI不知所措"

**→ 解决方案1（基于提示）**：分解为步骤

**修复**：
1. 阅读[高级技术：多步骤工作流程](./advanced-techniques-zh.md#多步骤工作流程)
2. 使用[提示链接](./advanced-techniques-zh.md#提示链接)
3. 在每一步验证

**→ 解决方案2（规范驱动）**：使用结构化规范

**修复**：
1. 使用[OpenSpec](../development-tools/recommended-tools/openspec-cli-zh.md)生成规范
2. 自动将功能分解为阶段和任务
3. 使用清晰的上下文为每个阶段单独提示AI
4. **最适合**：具有10+功能或复杂多组件系统的项目

## 提示 vs 规范：何时使用每个

**理解区别：**
- **提示** = 通信层（您如何与AI交谈）
- **规范** = 规划层（您正在构建什么）
- **结合** = 超级强大的vibecoding工作流程

### 决策矩阵

```
┌──────────────────────────────────────────────┐
│  项目类型 → 推荐方法                        │
├──────────────────────────────────────────────┤
│  Bug修复               → 仅提示              │
│  小功能（1-5h）  → 提示 + 模板           │
│  中等功能（5-20h）→ 提示 + 多步骤         │
│  大型项目（20h+）  → OpenSpec + 提示      │
│  团队协作    → 需要OpenSpec            │
└──────────────────────────────────────────────┘
```

### 规范驱动的工作流程

**当您使用[OpenSpec](../development-tools/recommended-tools/openspec-cli-zh.md)时：**

1. **规划阶段**：OpenSpec将您的PRD转换为结构化规范
   - 生成分阶段实施计划
   - 创建任务分解
   - 记录架构决策

2. **实施阶段**：使用此提示指南执行每个阶段
   - 将阶段规范加载到上下文中
   - 应用[功能实施模板](./template-library-zh.md#功能实施模板)
   - 对复杂任务使用[多步骤工作流程](./advanced-techniques-zh.md#多步骤工作流程)

3. **质量阶段**：结合两者进行验证
   - 根据规范验收标准验证实施
   - 使用[自我纠正提示](./advanced-techniques-zh.md#自我纠正提示)
   - 根据需要在阶段之间迭代

**结果**：结构化项目规划 + 质量AI通信 = 可靠、可扩展的开发

## 与Vibecoding工作流程集成

### 阶段0：Vibecoder准备
- 在开始项目之前审查[基础](./foundations-zh.md)
- 使用项目约定设置[.cursorrules](./advanced-techniques-zh.md#策略2有效使用cursorrules)

### 阶段1：规划

**根据项目复杂性选择您的方法：**

**方法1：仅提示（小型项目）**
- 使用提示生成PRD并直接开始编码
- 引用[文档模板](./template-library-zh.md#文档模板)
- 最适合：快速功能、1-5个任务、单独开发

**方法2：规范驱动（大型项目）⭐**
- 使用[OpenSpec](../development-tools/recommended-tools/openspec-cli-zh.md)将PRD转换为结构化规范
- OpenSpec将功能分解为阶段并生成markdown文档
- 然后应用提示模板实施每个阶段
- 最适合：复杂功能、10+个任务、团队项目

**强大的组合**：OpenSpec提供结构 → 提示确保质量实施

### 阶段2：开发
- 应用[功能实施模式](./task-specific-patterns-zh.md#功能实施)
- 使用[功能实施模板](./template-library-zh.md#功能实施模板)
- 对复杂功能使用[多步骤工作流程](./advanced-techniques-zh.md#多步骤工作流程)
- 遵循[模板库](./template-library-zh.md)以保持一致性

### 阶段3：测试和调试
- 使用[调试模式](./task-specific-patterns-zh.md#调试--错误解决)
- 使用[调试模板](./template-library-zh.md#调试模板)
- 应用[测试模板](./template-library-zh.md#测试模板)
- 遵循[错误恢复提示](./advanced-techniques-zh.md#错误恢复提示)
