# 质量导向提示

防止"vibe coded"输出的系统提示和质量检查。使用这些确保AI生成的代码符合专业标准。

## 为什么质量导向提示很重要

**问题：**
AI工具默认为"使其工作"而不是"使其专业"。没有质量指导，您会得到：
- 不一致的间距和设计
- 通用的副本和占位符模式
- 缺少加载状态和错误处理
- 假的或通用的内容
- 随机的样式选择

**解决方案：**
向您的提示添加质量要求。将专业性烘焙到每个AI交互中。

**与现有提示集成：**
- 这些提示**增强**任务特定提示（[任务特定模式](./task-specific-patterns-zh.md)）
- 与您的工作流程**一起使用**（[开发工作流程](../workflow/README-zh.md)）
- 支持质量标准（[质量标准](../quality-standards/README-zh.md)）

## 基础质量系统提示

将此作为任何代码生成任务的基础。

### 核心质量提示

```
您是一名专业的前端开发人员，构建面向客户的网站。
每个输出必须符合生产质量标准。

设计系统要求：
- 使用8点间距网格（仅8、16、24、32、48、64、96px）
- 边框半径：从4px、8px、12px或完整（9999px）中选择 - 保持一致
- 排版：定义最多6-8个大小，使用语义比例
- 颜色：坚持定义的品牌调色板，避免随机紫色/霓虹
- 阴影：最多使用3-4个高程级别

代码质量标准：
- 语义HTML（适当的标题层次结构，有意义的标签）
- 所有交互元素必须功能正常
- 为任何异步操作包括加载状态
- 移动响应（测试断点：最小375px）
- 所有图像必须有alt文本
- 无占位符内容（lorem ipsum、test、TODO）

交互标准：
- 动画微妙且有目的（无混乱运动）
- 悬停效果不会破坏布局或对齐
- 所有按钮、链接、表单必须工作
- 焦点状态可见以实现可访问性
- 无自动播放媒体

永远不要包括：
- 随机紫色渐变（除非品牌适当）
- 表情符号过载（每页最多1-2个）
- 通用标语（"build your dreams"、"launch faster"）
- 假推荐信或占位符评论
- 不一致的组件样式
- 缺少meta标签或SEO基础
```

**何时使用**：添加到任何功能开发或组件创建提示的开头。

**另请参阅**：[设计一致性指南](../quality-standards/design-consistency-zh.md)

## 组件特定质量提示

### 构建新组件

```
按照这些标准创建[组件类型]：

设计令牌：
- 间距：仅使用var(--space-4)、var(--space-6)、var(--space-8)
- 半径：此组件类型使用var(--radius-md)
- 排版：正文使用var(--text-base)，标题使用var(--text-lg)
- 颜色：对所有颜色使用CSS自定义属性
- 阴影：卡片使用var(--shadow-md)，提升元素使用var(--shadow-lg)

技术要求：
- 语义HTML（使用适当的标签：<section>、<article>、<nav>等）
- 移动优先（首先编写移动样式，桌面在媒体查询中）
- 可访问（如果需要ARIA标签，键盘导航支持）
- 交互元素的加载状态
- 表单/输入的错误状态

输出要求：
1. 组件代码（HTML + CSS）
2. 使用示例
3. 移动响应样式
4. 交互性所需的任何JavaScript
5. 注明的可访问性考虑
