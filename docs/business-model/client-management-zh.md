# 客户管理指南

以诚实、透明和清晰的沟通管理客户关系是可持续自由职业vibecoding工作的基础。本指南涵盖了从初始联系到长期维护的完整客户生命周期。

## 沟通策略

### 基于电子邮件的沟通
- **主要渠道**：电子邮件用于所有项目沟通
- **好处**：书面记录、异步、专业
- **响应时间**：24小时内回复询问
- **有组织的线程**：每个项目主题一个电子邮件线程

### 提前设定期望
在任何工作开始之前，建立：
- 您的网站可以做什么
- 它们不能做什么
- 您的技术方法
- 时间线期望
- 定价结构
- 沟通规范

### 诚实披露哲学
**核心原则**：透明建立信任，信任产生推荐。

- 解释您使用AI辅助开发
- 对能力和限制坦诚
- 不要过度承诺以赢得项目
- 如果您不是合适的人选，请说明

## 初始客户咨询

### 发现电话/电子邮件检查清单

**客户信息：**
- [ ] 业务类型和行业
- [ ] 当前网站情况（无、WordPress、过时等）
- [ ] 目标受众（本地、区域、国家）
- [ ] 设备使用（B2B桌面为主 vs B2C移动为主）
- [ ] 他们欣赏的竞争对手

**项目要求：**
- [ ] 网站目的（信息、潜在客户生成、电子商务）
- [ ] 所需页面（主页、关于、服务、联系等）
- [ ] 特殊功能（表单、画廊、预订等）
- [ ] 内容状态（就绪、需要创建、需要照片）
- [ ] 域名和电子邮件状态

**时间线和预算：**
- [ ] 期望的发布日期
- [ ] 预算范围
- [ ] 紧急程度

### 向客户提出的问题

**了解他们的业务：**
1. "您的业务为客户解决了什么问题？"
2. "什么使您与竞争对手不同？"
3. "您希望访问者在您的网站上采取什么行动？"
4. "您有品牌指南（颜色、字体、风格）吗？"

**了解他们的受众：**
1. "您的典型客户是谁？"
2. "大多数人目前如何找到您？"
3. "您的客户主要使用移动设备还是桌面设备？" ⭐
4. "客户经常问什么问题？"

**技术理解：**
1. "您目前有网站吗？您喜欢/不喜欢它什么？"
2. "您需要定期更新内容吗？"
3. "您有可以处理简单更新的人吗？"
4. "您的托管预算（如果有）是多少？"

### 何时拒绝项目

**危险信号：**
- 不现实的时间线（例如，"明天需要"）
- 不清楚的要求，带有"我们会弄清楚"
- 预算不匹配（想要$200的$5k网站）
- 过度的修订请求
- 不尊重的沟通
- 需要您专业知识之外的技术
- 期望24/7可用性
- 不会提供必要的材料（内容、图像）

**如何礼貌拒绝：**
```
"感谢您考虑我为您的项目。在审查您的要求后，
我认为我不是[具体原因]的最佳选择。我建议[替代建议或推荐，如果可能]。
感谢您联系我，我祝您的项目成功。"
```

### 相互决策过程

**这是一个双向评估：**
- 您评估项目是否适合您的技能
- 客户评估您的方法是否适合他们的需求
- 双方必须同意继续

**我的方法：**
"我解释我的整个工作流程：我如何工作，我使用什么工具，网站能做什么，重要的是，它们不能做什么。然后我问：'这种方法对您有效吗？'这不仅仅是我的决定——这是我们共同的决定。"

**示例脚本：**
```
"这是我的工作方式：我使用[Astro和Tailwind]构建现代静态网站。它们极其快速，在Google的性能指标上得分完美，并在Cloudflare上免费托管。

然而，这些网站与WordPress不同：
- 没有用于内容编辑的管理面板（您将通过电子邮件向我发送更改）
- 不适合有100+产品的电子商务
- 不是复杂Web应用程序的解决方案

对于[管道工/电工/理发师]业务网站，这种方法是完美的：快速、可靠、低维护和成本效益。

这种方法是否符合您的需求？"
```

## 解释您的工作流程

### 如何解释AI辅助开发

**选项1：完全披露（推荐）**
"我使用现代AI辅助开发工具更高效地构建网站。这意味着我可以更快、更具成本效益地交付，同时保持高质量。AI帮助代码生成，但我监督所有决策，审查所有输出，并确保一切都符合专业标准。"

**选项2：一般披露**
"我使用尖端开发工具和工作流程，使我能够高效且经济实惠地交付高质量网站。"

**选项3：不披露**
不推荐用于长期客户关系。透明建立信任。

### 您的网站能做什么

**清晰传达能力：**
- 专业、现代设计
- 移动响应式（在手机、平板电脑、桌面上完美工作）
- 闪电般快速加载（比95%的网站更好）
- 带有电子邮件通知的联系表单
- 照片画廊和作品集
- Google地图集成
- 社交媒体集成
- 博客功能（如果需要）
- SEO优化
- 可访问性合规
- 免费托管（Cloudflare Pages）
- HTTPS安全

### 您的网站不能做什么

**对限制诚实：**
- ❌ 不适合客户可编辑的CMS（没有像WordPress这样的管理面板）
- ❌ 不适用于大型电子商务（100+产品）
- ❌ 不适用于复杂Web应用程序（社交网络、SaaS平台）
- ❌ 不适用于客户需要每日内容更新的网站
- ❌ 没有内置预订/预订系统（虽然可以集成外部工具）

**如何定位限制：**
"对于您的用例[小型企业网站]，这些限制不适用。您不是经营有500产品的在线商店或每日发布博客文章。您需要一个专业网络存在，加载快速，看起来很棒，工作可靠——这正是这种方法提供的。"

### 为非技术客户解释技术堆栈

**避免行话。使用类比：**

**而不是：**
"我使用Astro构建网站，它使用岛屿架构进行部分水合，用Vite编译，并通过CI/CD部署到Cloudflare的边缘网络。"

**说：**
"我使用现代Web技术预构建您的整个网站，因此它为访问者即时加载。它托管在全球网络（Cloudflare）上，这意味着您的网站在世界任何地方都加载快速。托管是免费的，因为它是静态网站。"

**他们关心的关键点：**
- ✅ 快速（更好的Google排名）
- ✅ 可靠（没有像WordPress那样的更新停机时间）
- ✅ 安全（没有插件的黑客风险）
- ✅ 免费托管（每月节省$10-30）
- ✅ 现代和专业

## 定价透明度

### 清晰定价模式

**定价结构示例：**
```
基础商业网站：$300-500
- 3-5页（主页、关于、服务、联系）
- 联系表单
- 移动响应式设计
- Google地图集成
- 免费托管设置
- 时间线：5-7天

增强商业网站：$600-900
- 6-10页
- 照片画廊
- 服务展示页面
- 博客（如果需要）
- 社交媒体集成
- 增强SEO
- 时间线：10-14天

自定义报价：
- 更大项目
- 特殊功能
- 电子商务（小规模）
- 时间线：变化
```

### 无隐藏费用方法

**透明成本分解：**
```
网站开发：$500
域名（年度）：$12（客户直接支付）
托管：$0（Cloudflare Pages免费层）
电子邮件设置（可选）：$0-5/月（取决于提供商）

第一年总计：$512
持续年度成本：$12（仅域名）
```

**包含内容：**
- 所有开发工作
- 托管设置
- SSL证书（免费包含）
- 移动优化
- SEO基础
- 测试和发布
- 30天的小调整

**不包含内容：**
- 域名注册（客户直接购买）
- 内容写作（客户提供，或单独费用）
- 专业摄影（客户提供，或单独费用）
- 持续维护（见[维护指南](./maintenance-zh.md)）

**获得书面批准：**
"在我开始之前，请审查上述范围并确认这符合您的期望。"

### 付款条款和里程碑

**小型项目（$300-700）：**
- 50%预付
- 50%完成时

**大型项目（$700+）：**
- 33%预付
- 33%中点里程碑
- 34%完成时

**付款方式：**
- 银行转账（首选）
- PayPal/Stripe（如果需要）
- 支票（本地客户）

**退款政策：**
"如果您对初始设计方向不满意，我将退还预付款的50%。一旦开发进展超过初始批准，已完成的工作不可退款，但我将与您合作以达到满意。"

### 交付物定义

**具体：**
- "主页" ✅
- "美丽的主页" ❌（主观）
- "画廊中最多20张图像" ✅
- "很多照片" ❌（模糊）

### 时间线期望

**现实估计：**
- 简单网站（3-5页）：5-7天
- 标准网站（6-8页）：10-14天
- 复杂网站（10+页）：3-4周

**添加缓冲时间：**
- 您的估计 + 30% = 客户时间线
- 考虑修订、客户延迟、意外问题

**依赖关系：**
"时间线假设在[日期]前提供内容和图像。材料延迟将按比例延长时间线。"

### 修订政策

**包含修订：**
- 2轮全面设计/布局修订
- 开发期间无限小的文本/图像更新
- 错误修复（始终免费）

**额外修订：**
"超出2轮的额外重大设计更改将根据范围单独报价。"

**什么算作修订：**
- 第1轮："将配色方案从蓝色更改为绿色" ✅（包含）
- 第2轮："重新组织主页布局" ✅（包含）
- 第3轮："完全不同的设计方向" ⚠️（额外收费）

## 开发期间

### 更新频率和格式

**每周电子邮件更新：**
```
主题：[项目名称] - 第1周更新

本周进展：
✅ 主页设计完成
✅ 关于和服务页面进行中
🔄 等待：您的服务描述

下周：
- 完成剩余页面
- 集成联系表单
- 初始移动测试

请在[预览链接]审查主页

需要的问题/决定：
- 首选配色方案：选项A还是B？
- 画廊布局：网格还是砌体风格？

如果您有任何问题，请告诉我！
```

**里程碑检查：**
- 初始设计批准
- 内容集成完成
- 发布前审查
- 发布前最终批准

### 处理更改请求

**小更改（无影响）：**
"当然，我可以更改那个颜色/文本/图像。将包含在下次更新中。"

**中等更改（轻微影响）：**
"这是可行的。这将增加大约[X小时]，这意味着[对时间线的影响]和额外的$[金额]。您希望我继续吗？"

**大更改（范围蔓延）：**
"这与我们原始范围有显著不同。让我为该添加准备报价：
- 功能：[描述]
- 预计时间：[X小时]
- 额外成本：$[金额]
- 时间线影响：[X天]

您想现在添加这个，还是为将来的更新搁置？"

### 管理范围蔓延

**预防：**
- 详细范围文档
- "这是第2阶段的好主意"
- "让我们按原始范围发布，然后我们可以添加那个"

**当它发生时：**
1. 确认请求
2. 评估影响
3. 提出选项
4. 如果继续，获得书面批准

**脚本响应：**
"我想确保我们在同一页面上。这个添加[描述更改]超出了我们原始范围[引用协议]。我很高兴包含它，但它将[影响时间线/成本]。您希望如何进行？"

### 客户反馈循环

**请求具体反馈：**

**坏：**
"您觉得怎么样？"

**好：**
"请审查主页并提供关于以下方面的反馈：
1. 整体布局和结构
2. 配色方案（是否与您的品牌匹配？）
3. 图像和位置
4. 文本内容（需要任何更改吗？）
5. 缺少任何元素吗？

请在[日期]前回复，以便我们保持进度。"

**处理模糊反馈：**

**客户说：** "我不喜欢它。"

**您回应：**
"我想更好地了解您的担忧，以便我可以解决它们。您能帮我了解：
- 具体什么对您不起作用？
- 您希望看到什么代替？
- 有您喜欢的设计示例吗？
这将帮助我朝正确方向修改它。"

## 建立信任

### 诚实胜过短期收益

**诚实沟通的例子：**

**当您不能做某事时：**
"我没有该特定集成的经验。我可以学习它，但这需要额外时间，我不能保证时间线。我建议[替代解决方案]或为该组件找到专家。"

**当时间线有风险时：**
"我想坦诚：由于[原因]，我们落后于计划。当前完成估计是[新日期]。我为延迟道歉，我正在努力最小化进一步影响。"

**当您犯了错误时：**
"我在[发生了什么]方面犯了错误。我现在已经修复了它，并且我已经添加了[预防措施]以确保它不会再次发生。这是我的责任——没有额外费用。"

### 长期关系焦点

**投资客户成功：**
- 即使不产生收入也提供有用建议
- 当它们效果更好时推荐免费工具
- 适当时将客户推荐给专家
- 发布后跟进："网站表现如何？"

**保持联系：**
- 季度检查电子邮件
- 分享相关提示或更新
- 记住他们的业务里程碑
- 对他们的成功真正感兴趣

### 基于推荐的成长

**推荐循环：**
1. 交付优秀工作
2. 提供卓越服务
3. 偶尔超越期望
4. 客户告诉他人
5. 重复

**真实世界示例：**
"一个客户曾经恐慌地打电话，因为他们的域名过期（不是我的责任）。我帮助他们续订并重新连接一切——花了30分钟，没有收费。他们在接下来的2个月内给了我3个推荐。这些推荐产生了$1,800。那30分钟是我做过的最好的投资。"

**请求推荐：**
"我很高兴您对网站满意！如果您认识其他可能需要现代、快速网站的企业主，我将感激任何推荐。我的大部分工作来自像您这样的推荐。"

**不要：**
- 施压推荐
- 为推荐提供折扣（看起来绝望）
- 在项目完成前询问

**要做：**
- 使其容易："随时分享我的联系信息"
- 提供名片或传单
- 通过偶尔检查保持首要位置

## 真实世界场景

### 场景1：客户期望WordPress风格编辑

**客户：** "所以我可以登录并自己编辑内容，对吧？"

**您的回应：**
"这不是这些现代静态网站的工作方式——它们没有像WordPress这样的管理面板。相反，对于任何更新，您只需通过电子邮件向我发送更改，我将快速更新，通常在24小时内。

这种权衡是值得的：
- 您的网站加载速度快5-10倍
- 没有插件的安全漏洞
- 没有托管成本
- 没有更新破坏网站的情况

对于偶尔更新内容（不是每天）的[业务类型]，这种方法为您节省金钱和麻烦。大多数更新我花不到10分钟。

对于频繁更新（每日博客文章等），WordPress可能更适合。您期望什么样的更新频率？"

### 场景2：客户想要超出范围的功能

**客户：** "我们还可以添加预订系统和客户门户吗？"

**您的回应：**
"这些是很好的功能，但它们超出了我们当前项目的范围。以下是您的选项：

选项1：保持当前范围
- 按计划发布网站
- 在第2阶段添加预订系统
- 时间线：按计划在[日期]

选项2：扩展当前范围
- 现在添加预订系统
- 额外成本：$[金额]
- 新时间线：[日期 + X周]

选项3：第三方集成

## 关键要点

### 诚实原则
**诚实不仅仅是道德的——长期来看是有利可图的。**

- 短期：可能因诚实而失去项目
- 长期：建立声誉，获得推荐，晚上睡得好

### 相互决策框架
**您和客户都必须同意一起工作。**

不是："我希望他们选择我。"
但是："我们彼此适合吗？"

### 无隐藏费用方法
**透明比任何东西都更快建立信任。**

- 前期清晰定价
- 解释包含什么和不包含什么
- 没有意外收费
- 书面协议

### 长期游戏
**一个满意的客户 → 多个推荐 → 可持续业务**

- 偶尔超越期望
- 不收费修复您的错误
- 发布后保持联系
- 对他们的成功真正感兴趣
