# 客户管理指南

以诚实、透明和清晰的沟通管理客户关系是可持续自由职业vibecoding工作的基础。本指南涵盖了从初始联系到长期维护的完整客户生命周期。

## 沟通策略

### 基于电子邮件的沟通
- **主要渠道**：电子邮件用于所有项目沟通
- **好处**：书面记录、异步、专业
- **响应时间**：24小时内回复询问
- **有组织的线程**：每个项目主题一个电子邮件线程

### 提前设定期望
在任何工作开始之前，建立：
- 您的网站可以做什么
- 它们不能做什么
- 您的技术方法
- 时间线期望
- 定价结构
- 沟通规范

### 诚实披露哲学
**核心原则**：透明建立信任，信任产生推荐。

- 解释您使用AI辅助开发
- 对能力和限制坦诚
- 不要过度承诺以赢得项目
- 如果您不是合适的人选，请说明

## 初始客户咨询

### 发现电话/电子邮件检查清单

**客户信息：**
- [ ] 业务类型和行业
- [ ] 当前网站情况（无、WordPress、过时等）
- [ ] 目标受众（本地、区域、国家）
- [ ] 设备使用（B2B桌面为主 vs B2C移动为主）
- [ ] 他们欣赏的竞争对手

**项目要求：**
- [ ] 网站目的（信息、潜在客户生成、电子商务）
- [ ] 所需页面（主页、关于、服务、联系等）
- [ ] 特殊功能（表单、画廊、预订等）
- [ ] 内容状态（就绪、需要创建、需要照片）
- [ ] 域名和电子邮件状态

**时间线和预算：**
- [ ] 期望的发布日期
- [ ] 预算范围
- [ ] 紧急程度

### 向客户提出的问题

**了解他们的业务：**
1. "您的业务为客户解决了什么问题？"
2. "什么使您与竞争对手不同？"
3. "您希望访问者在您的网站上采取什么行动？"
4. "您有品牌指南（颜色、字体、风格）吗？"

**了解他们的受众：**
1. "您的典型客户是谁？"
2. "大多数人目前如何找到您？"
3. "您的客户主要使用移动设备还是桌面设备？" ⭐
4. "客户经常问什么问题？"

**技术理解：**
1. "您目前有网站吗？您喜欢/不喜欢它什么？"
2. "您需要定期更新内容吗？"
3. "您有可以处理简单更新的人吗？"
4. "您的托管预算（如果有）是多少？"

### 何时拒绝项目

**危险信号：**
- 不现实的时间线（例如，"明天需要"）
- 不清楚的要求，带有"我们会弄清楚"
- 预算不匹配（想要$200的$5k网站）
- 过度的修订请求
- 不尊重的沟通
- 需要您专业知识之外的技术
- 期望24/7可用性
- 不会提供必要的材料（内容、图像）

**如何礼貌拒绝：**
```
"感谢您考虑我为您的项目。在审查您的要求后，
我认为我不是[具体原因]的最佳选择。我建议[替代建议或推荐，如果可能]。
感谢您联系我，我祝您的项目成功。"
```

### 相互决策过程

**这是一个双向评估：**
- 您评估项目是否适合您的技能
- 客户评估您的方法是否适合他们的需求
- 双方必须同意继续
