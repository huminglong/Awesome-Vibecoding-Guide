# Kilo Code — 多功能IDE原生AI助手

## 概述

Kilo Code是一个多平台AI编码助手，提供VSCode插件和独立CLI。它对于喜欢在IDE内工作并具有视觉反馈，同时仍希望访问强大AI功能的开发人员特别有价值。双重方法策略提供了灵活性，可以选择您首选的环境而不牺牲功能。

## 为什么Kilo Code值得关注

### 最适合IDE偏好的开发人员

如果您更喜欢在VSCode内工作并具有视觉反馈、紧密的IDE集成和基于GUI的交互，Kilo Code值得认真考虑。它提供：

- **紧密的IDE集成**，具有调试和视觉反馈
- **VSCode原生体验**，无需切换到终端
- **CLI回退**，当您需要轻量级交互时
- **两种接口之间的无缝切换**

### 智能模式

Kilo的不同操作模式是其最强功能。而不是一刀切的交互，您可以为您的特定任务选择正确的模式：

- **代码模式**（默认）：具有完整工具访问的通用编码任务和开发
- **架构师模式**：规划、技术领导和系统设计，具有受限编辑（仅markdown）
- **询问模式**：回答问题和提供信息而不修改代码（只读）
- **调试模式**：系统问题诊断和故障排除，具有完整工具访问
- **编排器模式**：战略工作流程协调，分解复杂任务并委托给专业模式
- **自定义模式**：创建您自己的专业助手，具有自定义工具访问和行为

这种模式灵活性意味着您不会在过度思考简单任务上浪费令牌，也不会在复杂挑战上资源不足。

### 灵活的定价模型

使Kilo脱颖而出的是三重提供：

- **免费模型**：内置访问免费模型，无需您自己的密钥
- **通过Kilo按使用付费**：使用Kilo的托管模型，按使用付费，按确切的提供商费率，无加价
- **自带密钥（BYOK）**：从任何主要提供商（Anthropic、OpenAI、Google等）连接您自己的API密钥
- **提供商灵活性**：根据任务要求轻松在提供商之间切换

您可以完全免费开始以测试工作流程，使用Kilo的按使用付费服务以获得便利，或自带密钥以获得最大控制。无供应商锁定，无强制订阅。

### 通用提供商支持

Kilo支持几乎每个主要LLM提供商：
- Anthropic（Claude）
- OpenAI（GPT模型）
- Google（Gemini）
- 以及通过标准API接口的更多

### 理想用于：
- **IDE偏好的开发人员**，他们想要视觉反馈和紧密集成
- **成本意识的开发人员**，寻找灵活的定价选项（免费 → BYOK）
- **多提供商用户**，他们想要轻松在不同的LLM提供商之间切换
- **初学者**，从免费模型开始并升级到高级密钥
- **VSCode用户**，他们想要AI辅助而不离开他们的IDE
- **团队**，需要具有集中计费的企业功能

### 特别有价值当：
- 您更喜欢基于GUI的工作流程而不是CLI
- 您需要针对不同任务复杂性的智能模式选择
- 预算灵活性很重要（免费模型用于学习，BYOK用于生产）
- 您想要比较不同的LLM提供商而无需工具切换

## 限制和考虑

### 当前约束
- **较新的CLI**：与Claude Code或Droid等既定工具相比，CLI仍然相对较新
- **文档**：CLI文档随着工具成熟而发展
- **VSCode插件历史**：插件仍然与Cline共享一些架构遗产

### 不是最佳选择用于：
- 主要在终端/CLI工作流程中工作的开发人员
- 需要针对每个边缘情况的广泛经过战斗测试的文档的用户
- CLI速度最重要的重度vibecoding工作流程
