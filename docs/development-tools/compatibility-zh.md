# 工具兼容性 ⚙️

## 核心集成要求

本指南描述了工具如何与Awesome Vibecoding Guide方法兼容。推荐的工具子集（Zed、Droid CLI / Claude Code）旨在提供最佳性能，是作者的选择，但也可以使用其他工具。为了最佳兼容性，工具应该支持：

### 1. 自带密钥（BYOK）集成
- **外部LLM API支持**：对于成本效率和模型灵活性至关重要
- **自定义端点配置**：能够连接到您首选的LLM提供商
- **API密钥管理**：安全处理多个API密钥和提供商

### 2. MCP（模型上下文协议）服务器支持
- **外部源集成**：连接到文档、数据库和外部资源
- **开发效率**：对相关上下文和工具的简化访问
- **稳健性**：通过多个数据源增强可靠性

## 兼容工具

### 基于CLI的编码代理
- **Droid CLI**：具有BYOK支持的命令行界面
- **Claude Code**：Anthropic的具有API集成的CLI工具
- **Kilo CLI**：具有智能模式和BYOK支持的多平台助手（参见[推荐工具](./recommended-tools/kilocode-zh.md)）
- **AmpCode**：替代CLI编码助手（参见[荣誉提及](./honorable-mentions-zh.md)以获取免费层级）
- **Codex**：GitHub的编码助手工具
- **其他类似的CLI代理**：任何支持外部LLM API的工具

### 具有插件支持的IDE和编辑器
- **Zed**：快速、现代的代码编辑器，具有可扩展性
- **VS Code**：广泛的插件生态系统和MCP支持
- **Cursor**：具有外部模型支持的AI驱动IDE
- **Cursor AI**：高级AI集成功能

### AI助手插件
- **Kilo VSCode插件**：具有双CLI/IDE支持的AI编码助手（参见[推荐工具](./recommended-tools/kilocode-zh.md)）
- **Roo**：具有外部集成的开发助手
- **Cline**：用于开发的增强AI助手
- **Continue**：具有BYOK支持的开源AI助手

## 集成好处

### 成本效率
- **GLM集成**：在适当的地方使用高性价比模型
- **提供商灵活性**：根据任务复杂性在提供商之间切换
- **使用优化**：将请求路由到最具成本效益的模型

### 开发效率
- **上下文感知**：MCP服务器提供相关项目上下文
- **工具编排**：开发工具之间的无缝集成
- **实时辅助**：立即访问相关文档和资源

### 稳健性和可靠性
- **多个数据源**：减少对单一提供商的依赖
- **回退机制**：模型和数据访问的备份选项
- **可扩展性**：轻松添加新工具和集成

## 不推荐的组合

- **封闭平台**：Replit、Lovable、Bolt（有限的集成选项）
- **MCP服务器过载**：同时太多MCP服务器（性能影响）
- **锁定工具**：没有BYOK或外部API支持的平台
- **单一供应商依赖**：将您锁定到一个生态系统的工具

## 实施指南

1. **从核心工具开始**：从Droid CLI或Claude Code开始（或参见[免费替代方案](./honorable-mentions-zh.md)）
2. **逐步集成**：根据需要添加MCP服务器
3. **API管理**：设置安全的API密钥处理
4. **监控使用**：跟踪跨提供商的成本和性能
5. **灵活性优先**：选择支持多种集成方法的工具
