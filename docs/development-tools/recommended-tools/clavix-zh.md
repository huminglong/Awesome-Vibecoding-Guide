# Clavix — 基于CLEAR的PRD生成器 📋

Clavix是一个开源提示工程框架，将粗略的想法转化为完善的提示、完整的PRD（产品需求文档）和可实施的任务列表。建立在学术验证的CLEAR方法论之上，它弥合了初始概念和生产就绪实施之间的差距。

## 🎯 核心价值主张

### 基于CLEAR的方法

Clavix利用**CLEAR框架**进行系统提示优化：
- **C**oncise（简洁）——消除歧义和噪音
- **L**ogical（逻辑）——确保结构化、合理的流程
- **E**xplicit（明确）——使要求和期望非常清楚
- **A**daptive（自适应）——调整到上下文和特定需求
- **R**eflective（反思）——结合反馈和迭代

这种方法确保所有AI交互的一致性和质量，从简单提示到复杂项目规范。

### 无缝PRD生成

将模糊的想法转化为生产就绪的文档：

1. **粗略想法** → 用简单的语言描述您想要构建的内容
2. **CLEAR优化** → 框架完善和结构化您的输入
3. **完整PRD** → 获得全面的产品需求文档
4. **任务列表** → 具有明确验收标准的可实施任务
5. **进度跟踪** → 监控实施并标记任务完成

**结果**：从概念到可操作的实施计划，只需几分钟而不是几小时。

## 🚀 主要用例

### 1. 功能规划和规范

**工作流程：**
```
用户："我想添加具有社交登录的用户身份验证"

Clavix输出：
→ 使用CLEAR框架优化的提示
→ 具有范围、要求、约束的完整PRD
→ 按优先级分解的实施任务
→ 准备AI编码助手执行
```

### 2. 项目启动和文档

非常适合启动新项目或功能：
- **MVP规划** → 清楚地定义范围和要求
- **技术规范** → 记录架构和决策
- **任务分解** → 生成可操作的实施步骤
- **团队对齐** → 共享清晰、结构化的文档

### 3. AI助手的提示优化

增强您与AI编码工具的交互：
```bash
# 快速优化
/clavix:fast "add dark mode to dashboard"

# 深度分析和全面完善
/clavix:deep "build real-time collaborative editing"

# 完整PRD工作流程
/clavix:prd
```

### 4. 实施进度跟踪

内置任务管理（v2.8+）：
```bash
# 列出PRD中的所有任务
clavix tasks list

# 标记任务完成并自动提交
clavix task-complete <taskId>

# 查看项目进度
clavix tasks status
```

## 💡 主要功能

### 1. 双优化模式

- **快速模式** → 快速完善以供立即使用
- **深度模式** → 全面分析和重组

两种模式都自动将输出保存到`.clavix/outputs/prompts/`以进行版本控制和审查。

### 2. PRD生成引擎

完整的产品需求文档包括：
- **执行摘要** → 高级概述
- **范围定义** → 什么在范围内，什么不在
- **技术要求** → 详细规范
- **实施任务** → 准备执行的检查清单
- **验收标准** → 清晰的成功指标

### 3. 任务管理系统

无缝跟踪实施进度：
- **任务生成** → 从PRD自动创建任务
- **进度跟踪** → 监控完成状态
- **自动提交** → 标记任务完成时生成提交
- **状态更新** → 项目进度的实时视图

### 4. 提示生命周期管理（v2.7+）

智能提示存储和卫生：
- **状态跟踪** → NEW → EXECUTED → STALE
- **年龄警告** → >7天 = OLD，>30天 = STALE
- **有组织存储** → 在`.clavix/outputs/prompts/`中结构化
- **轻松检索** → 列出、查看和执行保存的提示

### 5. 通用AI代理集成

与15+编码助手无缝协作：
- **Claude Code**（通过斜杠命令）
- **Cursor**（内置支持）
- **Windsurf**（原生集成）
- **Droid CLI**（自定义指令）
- **任何AI代理**支持斜杠命令或基于文件的指令

## 🔧 实际实施

### 安装和设置

```bash
# 全局安装
npm install -g clavix

# 在您的项目中初始化
clavix init
```

**要求**：Node.js ≥ 16.0.0（ESM支持）

### 快速入门工作流程

1. **生成PRD**
   ```bash
   /clavix:prd
   # 或通过CLI：clavix prd
   ```

2. **审查和完善**
   - 在`.clavix/outputs/`中打开生成的PRD
   - 根据需要编辑以匹配您的要求
   - 与团队分享以对齐

3. **生成实施任务**
   ```bash
   clavix tasks generate
   ```

4. **使用AI助手执行**
   ```bash
   # 查看保存的提示
   clavix prompts list

   # 执行优化的提示
   /clavix:execute
   ```

5. **跟踪进度**
   ```bash
   # 标记任务完成（自动提交）
   clavix task-complete <taskId>

   # 查看状态
   clavix tasks status
   ```

### 斜杠命令

对于支持的AI代理（Claude Code、Cursor、Windsurf）：
- `/clavix:fast "description"` — 快速优化
- `/clavix:deep "description"` — 全面分析
- `/clavix:prd` — 完整PRD工作流程
- `/clavix:execute` — 运行保存的提示

### 最佳实践

1. **对复杂功能从深度模式开始**
   - 对新功能或重大更改使用`/clavix:deep`
   - 让CLEAR框架识别边缘情况和要求
   - 执行前审查和完善

2. **利用PRD生成**
   - 在项目开始时生成PRD以对齐
   - 在开发期间作为活文档使用
   - 随着要求的发展更新PRD

3. **维护任务卫生**
   - 完成任务时标记它们为完成
   - 使用自动提交功能以获得一致的历史记录
   - 定期审查任务状态

4. **版本控制您的.clavix目录**
   - 与代码一起跟踪PRD和提示
   - 在团队之间共享优化
   - 维护项目历史和决策

## 🆚 为什么选择Clavix而不是替代方案

### vs. OpenSpec CLI
- **更快** → CLEAR优化几乎是瞬间的
- **更简单** → 没有复杂的CLI命令需要记忆
- **集成** → 原生与15+ AI代理协作
- **任务管理** → 内置进度跟踪
- **免费和开源** → 无许可成本

### vs. 手动提示
- **一致性** → CLEAR框架确保质量
- **完整性** → 永不遗漏边缘情况或要求
- **效率** → 在几分钟而不是几小时内生成PRD
- **最佳实践** → 内置验证的方法论

### vs. 传统PRD工具
- **AI原生** → 为AI助手工作流程设计
- **可执行** → PRD自动成为实施任务
- **集成** → 工具之间无上下文切换
- **版本控制** → 您的存储库中的Markdown文件

Clavix通过结合学术严谨性（CLEAR框架）和实际实施（PRD + 任务列表），改变了您规划和执行开发工作的方式。不再为模糊的提示而挣扎或花费数小时编写规范，您获得优化的、可操作的文档，与您的AI编码工作流程无缝集成。结果：更快的规划、更清晰的要求和更可靠的实施——同时保持自然、对话式的vibecoding体验。
