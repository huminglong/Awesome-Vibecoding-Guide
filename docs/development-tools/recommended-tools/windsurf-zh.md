# Windsurf — 高速AI IDE

## 概述
Windsurf是Codeium的AI驱动IDE（2025年被Cognition收购），作为VS Code分支构建，具有集成的代理功能。它以其SWE-1.5代理模型脱颖而出，达到每秒950个令牌——比Claude Sonnet 4.5快13倍——同时保持接近前沿的编码性能。

## 主要优势

### SWE-1.5：速度无妥协
- **每秒950个令牌**：比Sonnet 4.5（69 tok/s）快13倍，比Haiku 4.5快6倍
- **在SWE-Bench Pro上达到40.08%**：接近SOTA性能（与Sonnet 4.5的43.60%相比）
- **前沿规模模型**：数百亿个参数针对速度进行了优化
- **协同设计架构**：模型、推理系统和代理工具为性能统一

### Codemaps：可视化代码库理解
- **AI注释的代码地图**：由SWE-1.5和Sonnet 4.5驱动
- **可视化导航**：查看跨模块的连接、流程和关系
- **即时跳转**：单击任何节点直接导航到代码位置
- **智能或快速模式**：在彻底分析（Sonnet 4.5）或快速映射（SWE-1.5）之间选择
- **最适合**：入职新代码库、跟踪客户端-服务器流程、调试身份验证

### 开发体验
- **VS Code分支**：与扩展和熟悉工作流程的完全兼容性
- **Cascade集成**：具有上下文感知建议的内置AI聊天
- **计划模式**：用于复杂任务的原生协作规划
- **浏览器集成**：来自Web资源的隐式上下文理解
- **积极开发**：频繁更新（Wave 10、Wave 11）具有新功能

## 最佳用例
- 想要尽可能快的AI代理响应的开发人员（950 tok/s）
- 寻求集成AI而无需切换IDE的VS Code用户
- 入职大型代码库的团队（Codemaps可视化）
- 速度比无限请求更重要的快速原型设计
- 需要可视化代码流理解的项目

## 限制和权衡

### 定价模型
- **每月$15获得500次请求**：月度上限可能限制重度用户
- **无BYOK支持**：不能像Zed那样自带API密钥（除了Anthropic API密钥）
- **与替代方案比较**：
  - Zed（免费）+ GLM（$3-6）= 具有滚动窗口的无限制请求
  - Windsurf = 更快的体验但固定的月度限制

### 何时选择替代方案
- **使用Zed + Droid CLI**如果您需要：
  - 无限制使用而没有请求上限
  - BYOK灵活性（使用任何LLM提供商）
  - 最大成本优化
  - 更轻的资源使用（Zed使用VS Code资源的20%）

- **使用Windsurf**如果您优先考虑：
  - 绝对速度（950 tok/s代理）
  - VS Code生态系统兼容性
  - 无需配置的一体化解决方案
  - 可视化代码库探索（Codemaps）

## 技术细节

### 性能基准
- **SWE-1.5速度**：通过Cerebras合作伙伴关系每秒950个令牌
- **延迟优化**：每步减少开销高达2秒
- **代理效率**：重写了lint检查和命令执行管道

### 模型访问
- 免费层级可用于实验
- 付费层级：每月$15获得500次请求，跨多个模型
- 支持OpenAI、Anthropic和专有SWE模型

## 为什么重要
Windsurf代表了AI辅助开发的有意义演变：它证明了速度和智能不必权衡。SWE-1.5模型的950 tok/s吞吐量使AI编码感觉真正即时，而Codemaps解决了一个真正的痛点——理解不熟悉的代码库。

对于VS Code用户，Windsurf提供了无需学习新IDE的高速vibecoding的最平滑路径。权衡是灵活性：您被锁定在Windsurf的定价模型中，而不是通过BYOK提供商优化成本。

### 与Zed互补
Windsurf和Zed服务于不同的需求：
- **Windsurf**：速度优先、一体化、VS Code熟悉度
- **Zed**：灵活性优先、BYOK、资源效率、无限制使用

两者都是合理的选择，取决于您重视最大速度（Windsurf）还是最大控制（Zed）。
